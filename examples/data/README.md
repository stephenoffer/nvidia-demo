# Test Data Files

This directory contains small test files for validating datasource implementations.

## Generated Test Files

All test files are small (< 1KB) and generated programmatically:

- `test_groot.jsonl` - GR00T format JSONL file (3 episodes)
- `test_binary.bin` - Structured binary file (5 records)
- `test_config.yaml` - YAML robot configuration
- `test_calibration.yaml` - Camera calibration data
- `test_robot.urdf` - URDF robot description
- `test_archive.zip` - ZIP archive with 3 files
- `test_data.msgpack` - MessagePack serialized data (2 records)
- `test_data.h5` - HDF5 file with 2 datasets
- `test_pointcloud.pcd` - PCD point cloud (5 points)
- `test_pointcloud.ply` - PLY point cloud (5 points)
- `test_data.pb` - Protobuf text format file

## Formats Requiring External Test Files

The following datasources require actual data files that cannot be easily generated:

- **MCAP** (`.mcap`) - ROS2 MCAP format, requires MCAP library
- **ROS Bag** (`.bag`) - ROS1 bag files, requires rosbag library
- **ROS2 Bag** (`.db3`) - ROS2 bag files, requires rosbag2 library
- **Velodyne** (`.pcap`, `velodyne*.bin`) - LIDAR packet capture files

To test these datasources, place sample files in this directory with the appropriate extensions.

## Regenerating Test Files

Test files can be regenerated by running the datasource test functions:

```bash
python3 pipeline/datasources/yaml_config.py
python3 pipeline/datasources/binary.py
# etc.
```

Each test function creates its required test file, runs the datasource, and cleans up afterward.
